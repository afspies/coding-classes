    \section{Modules and Packages}\label{modules-and-packages}

We don't want to reinvent the wheel - If someone has implemented
something for us, then we can just use that.

    \begin{itemize}
\tightlist
\item
  Modules are collections of functions which somebody else has built
\end{itemize}

    \begin{itemize}
\tightlist
\item
  Packages are nicely wrapped up collections of modules
\end{itemize}

    \subsection{Import}\label{import}

The way we load a module into our code is with the \textbf{import}
statement, e.g.:

    \begin{Verbatim}[commandchars=\\\{\}]
{\color{incolor}In [{\color{incolor}5}]:} \PY{c+c1}{\PYZsh{} We snuck this in a while ago}
        \PY{c+c1}{\PYZsh{} \PYZdq{}time\PYZdq{} is a package which comes with python}
        
        \PY{k+kn}{import} \PY{n+nn}{time}   \PY{c+c1}{\PYZsh{} We import the time module}
        \PY{n}{time}\PY{o}{.}\PY{n}{sleep}\PY{p}{(}\PY{l+m+mi}{3}\PY{p}{)} \PY{c+c1}{\PYZsh{} sleep function is part of time module}
\end{Verbatim}

    \subsection{From}\label{from}

Sometimes we don't want to import everything from a module - we can pick
using \textbf{from}

    \begin{itemize}
\tightlist
\item
  E.g. if we have a function sleep(), we probably don't want to import
  the sleep() function from time
\end{itemize}

    Additionally, if we use from, we don't need to explicitly call the
function via the module

    \begin{Verbatim}[commandchars=\\\{\}]
{\color{incolor}In [{\color{incolor}10}]:} \PY{c+c1}{\PYZsh{} All I want to do is sleep}
         \PY{k+kn}{from} \PY{n+nn}{time} \PY{k}{import} \PY{n}{sleep}
         \PY{n}{sleep}\PY{p}{(}\PY{l+m+mi}{3}\PY{p}{)}  \PY{c+c1}{\PYZsh{} Don\PYZsq{}t need to call via time module}
\end{Verbatim}

    \subsection{As}\label{as}

If we want to refer to the imported functions using a different name,
then we can import them \textbf{as} something else

    \begin{Verbatim}[commandchars=\\\{\}]
{\color{incolor}In [{\color{incolor}12}]:} \PY{c+c1}{\PYZsh{} Zen}
         \PY{k+kn}{from} \PY{n+nn}{time} \PY{k}{import} \PY{n}{sleep} \PY{k}{as} \PY{n}{meditate}
         \PY{n}{meditate}\PY{p}{(}\PY{l+m+mi}{3}\PY{p}{)}
\end{Verbatim}

    This can be useful if you have a function with the same name, or if the
name of the module/function is annoyingly long

    \begin{Verbatim}[commandchars=\\\{\}]
{\color{incolor}In [{\color{incolor} }]:} \PY{k+kn}{import} \PY{n+nn}{time} \PY{k}{as} \PY{n+nn}{t} \PY{c+c1}{\PYZsh{} Can change module name}
        \PY{n}{t}\PY{o}{.}\PY{n}{sleep}\PY{p}{(}\PY{l+m+mi}{3}\PY{p}{)}
\end{Verbatim}

    \section{Coding in the real world}\label{coding-in-the-real-world}

Most new code is written to solve new problems. It is rare that these
problems don't build on previous work

    In reality, much of your time will probably be spent figuring out how to
repurpose code written by strangers, or colleagues, in order to solve
the task in front of you.

    If you're lucky, there will be thorough \textbf{documentation} for the
package / modules in question (and \emph{good comments!})

    \section{Turtle}\label{turtle}

In the exercises you'll be playing around with numpy and turtle. You
will make use of the documentation! :O

    Turtle is a very basic animation module that comes with python:

    \begin{itemize}
\tightlist
\item You tell a turtle to move, and it draws a line behind it
\item The turtle can also fill in regions using \textbf{begin\_fill()} and
  \textbf{end\_fill()} {[}see example{]}

       \section{Additional Stuff}\label{additional-stuff}

\subsection{Using the terminal}\label{using-the-terminal}

\textbf{This makes you look like a pro-hacker}

Most professional programming is done on linux or mac, both of which are
based on UNIX. Windows users can install the unix subsystem with fairly
little effort (I recommend looking at
https://www.windowscentral.com/how-install-bash-shell-command-line-windows-10).

"pip install" is an example of a terminal command, but there are many
more. Amongst these, the most commonly used are those which allow you to
move around inside your PCs directories:

\begin{itemize}
\tightlist
\item
  \textbf{ls} lists the files and folders in your current directory, and
  \textbf{ls -a} includes additional information

  \begin{itemize}
  \tightlist
  \item
    Here -a is an example of an additional parameter. Many commands have
    these, and you can typically find out about the possible
    parameters/usage of a command by using the -\/-help tag. For
    example, "ls -\/-help"
  \end{itemize}
\item
  \textbf{cd} short for change directory allows you to move between
  folders

  \begin{itemize}
  \tightlist
  \item
    e.g. "cd my\_folder* moves you into the folder my\_folder, if it
    exists in the current directory
  \end{itemize}
\item
  \textbf{pwd} prints the working directory, so you know where you are
  in the filesystem
\item
  \textbf{mkdir} creates a directory. E.g. "mkdir fish" creates a folder
  named "fish" in your current directory
\item
  \textbf{less} allows you to view the contents of a file without
  opening it. You can exit this using the "q" key.

  \begin{itemize}
  \tightlist
  \item
    e.g. if there's a "test.txt" file, you can use "less.txt" to
    directly view its contents in the terminal window
  \end{itemize}
\item
  \textbf{rm} removes stuff - \textbf{be careful with this}. The
  \textbf{-r} parameter removes recursively, allowing you to delete
  folders and their contents

  \begin{itemize}
  \tightlist
  \item
    e.g. "rm -r my\_folder" deletes the folder "my\_folder"
  \item
    "sudo rm -rf /*" deletes everything on your pc! :D (the -f parameter
    forces deletion of protected files)
  \end{itemize}
\end{itemize}
