\subsection{Exc. 1 (and): Multiple Users}
 (This
exercise is similar to the \emph{extra} part of last week's summary.)

Write some code which asks a user to "login". This means, they will need
to provide a correct username \textbf{and} password. * You should have
multiple possible username-password combinations which can be "logged
into" * Usernames should \emph{not} be sensitive to capital letters

    \subsection{Exc. 2 (lists \& in): Bad Password
Warning}\label{exc.-2-lists-in-bad-password-warning}

Let's make our system more slick by using what we know about lists.

\textbf{Note:} Add this functionality to your solution for the previous
exercise.

\subsubsection{ 2.a Check for bad passwords}
Create a list containing what you'd consider bad passwords (e.g. "password", "1234" etc.). 
When a user has succesfully logged in, if their password is a poor one, then tell them so

\subsubsection{2.b}\label{b}

Store user details in lists. * Create two lists, one containing
usernames and the other containing passwords * A user logs in
succesfully if they enter a username-password pair corresponding to the
i\emph{th} usernames in the username list, and the i\emph{th} password
in the passwords list

\subsubsection{2.c}\label{c}

Reset bad passwords * If a user logs in and their password is identified
as a bad password, give them the \textbf{option} to change it (i.e.
change the appropriate entry in the passwords list)

These challenges are tricky and require lots of clever steps! Don't feel

    \subsection{Challenge 1 (Modulu) - Fizz
Buzz}\label{challenge-1-modulu---fizz-buzz}

This is a classic problem which you'll probably encounter many times if
you learn other programming languages in future.
\subsubsection{Problem Statement}
Given an integer N (provided by the user), print out all the
integers from 0 to N. However, you must replace numbers as follows: * If
a number is a multiple of 3, then print \textbf{Fizz} instead of the
number * If a number is a multiple of 5, then print \textbf{Buzz}
instead of the number * If a number is a multiple of 3 \textbf{and} 5,
then print \textbf{Fizz Buzz}

\subsubsection{Extension}\label{extension}

\begin{itemize}
\tightlist
\item
  Instead of using the built-in \% operator from python, implement it
  yourself using another while loop - Can't trust the python developers!
\item
  Make sure that your program can handle \textbf{edge cases} - This is
  extremely important for real programmes
\end{itemize}

    \subsection{Challenge 2 (While + List Slicing) - Pascal's
Triangle}\label{challenge-2-while-list-slicing---pascals-triangle}

Here you will create a piece of code which generates Pascal's triangle
\subsubsection{Base Problem Statement}
Given an integer N (provided by the user),
using a while loop print out Pascal's Triangle in the format: 1, 121,
1331, 14641,... row\_N\\
{[}Try doing this without creating more than one list per loop step for more difficulty{]}

\subsubsection{Extra Challenge}\label{extra-challenge}

\begin{itemize}
\tightlist
\item
  Use \emph{Nested Lists}! Store each row as a list inside of a larger
  list, e.g.: {[}{[}1{]}, {[}121{]}, {[}1331{]},... {[}row\_N{]}{]}
\item
  After you've created your pascal's triangle list, print it out so it
  looks like an actual triangle:\\
  \_\_1\_\_$\\
  \_1\_2\_1 ...
\end{itemize}
